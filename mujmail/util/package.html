Provides functions that are used in whole application. Classes in this package can be divided into five groups:

  • Class that provides decoding fu
  • Classes that provides decoding and generic purpose functions and .
  • Classes provides functionality of observing objects.
  • Classes provides functionality of buffering.
  • Classes provides functionality of persistent saving and loading variables.
  • Interfaces used in whole application.

Decoding and generic purpose functions

Decoding functions provides class {@link mujmail.util.Decode}. Generic purpose functions are contained in class {@link mujmail.util.Functions}.

Observing objects

Observing objects can be used for implementation of model-view paradigm. The object that should be observable have to be subclass of {@link mujmail.util.Observable}. Objects that want to receive notifications when observable object is changed have to implement interface {@link mujmail.util.Observer} and have to register itself to observable object using methods of class {@link mujmail.util.Observable}.

Buffering

If it is necessary to write some data not immediately, but later, class {@link mujmail.util.OutputBuffer} can be subclassed to implement such functionality. Class {@link mujmail.util.OutputBuffer} provides functions to write data from the buffer if the seize of data is bigger than gven limit or when it is out of the memory. There are yet implemented buffers for various types of data - for instance {@link mujmail.util.OutputBuffer.OutputStreamBuffer} that can be used for writing array of bytes into stream.

Persistent saving and loading of variables

For saving variables to persistent storage and loading it there is an abstract class {@link mujmail.util.ValuePersistentReminder}. It's implementation {@link mujmail.util.ValuePersistentReminder.BooleanValuePersistentReminder} can be used for saving and loading boolean variables.
