Classes in this package can be divided into this groups:

  • Main class of the application.
  • Classes representing an email.
  • Classes used to store the content of emails in persistent storage.
  • Classes representing boxes that contain emails.
  • Classes containing settings and properties of the application.
  • Class used for localization.
  • Classes representing user interface components for sending mails.
  • Classes representing other user interface components such as address book or menu.
  • Classes used for exporting mails and bodyparts to the filesystem.
  • Class holding all ever seen mails.
  • Other classes.

Representing an email

An email is represented by class {@link mujmail.MessageHeader}. This class contains information contained in a header of the email and list of body parts - email body and email attachments. Body part is represented by class {@link mujmail.BodyPart}. It contains information about the body part and object of class {@link mujmail.ContentStorage} that stores the content of the storage. All this information are stored persistently in RMS database.

Storign the content of an email in persistent storage

The content of a body part of an email is represented by abstract class {@link mujMailContentStorage}. There are now two implementation of this class. {@link mujmail.RMSStorage} is used to store the content of body part in RMS database. It uses class {@link mujmail.MailDB} for saving and loading fragments of the content from RMS database. {@link mujmail.FSStorage} is used to store the content of body part in file in file system of the phone. It does not support writing new content to the storage yet so it is now used only to represent body parts that user attached to the mail when writing an email.
